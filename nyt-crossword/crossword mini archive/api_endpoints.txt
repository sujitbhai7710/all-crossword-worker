NYT MINI CROSSWORD ARCHIVE API DOCUMENTATION

Base URL: https://nyt-mini-archive.nytsolver.workers.dev

================================================================
PUBLIC ENDPOINTS (GET)
================================================================

1. Get Today's Puzzle
   URL: /today
   Description: Returns the most recent puzzle stored in the database.
   Example: https://nyt-mini-archive.nytsolver.workers.dev/today

2. Get Puzzle by Date
   URL: /date?date=YYYY-MM-DD
   Description: Returns the structured puzzle data for a specific date.
   Example: https://nyt-mini-archive.nytsolver.workers.dev/date?date=2026-01-01

3. Get Formatted Text
   URL: /formatted?date=YYYY-MM-DD
   Description: Returns the clues and answers in a plain text format (ACROSS/DOWN blocks).
   Example: https://nyt-mini-archive.nytsolver.workers.dev/formatted?date=2026-01-01

4. List Available Dates (Pagination)
   URL: /list?page=1&limit=50
   Description: Returns a paginated list of all dates available in the archive, sorted from newest to oldest. 
   Parameters:
     - page: Page number (default: 1)
     - limit: Number of items per page (default: 50, max: 100)
   Example: https://nyt-mini-archive.nytsolver.workers.dev/list?page=1&limit=50

5. Search by Clue
   URL: /clue?q=search_term
   Description: Searches all archived puzzles for clues matching the query.
   Example: https://nyt-mini-archive.nytsolver.workers.dev/clue?q=music

6. Search by Answer
   URL: /answer?q=search_term
   Description: Searches all archived puzzles for answers matching the query.
   Example: https://nyt-mini-archive.nytsolver.workers.dev/answer?q=PIANO

================================================================
ADMIN / UPDATE ENDPOINTS (GET)
================================================================
Note: These require your secret API_KEY (currently: BloggingIo@7)

1. Manual Fetch Today
   URL: /today/add/{API_KEY}
   Description: Forces the worker to fetch today's puzzle from NYT and store it.
   Example: https://nyt-mini-archive.nytsolver.workers.dev/today/add/BloggingIo@7

2. Manual Fetch by Date
   URL: /date/add/{API_KEY}?date=YYYY-MM-DD
   Description: Forces the worker to fetch and store a puzzle for a specific date.
   Example: https://nyt-mini-archive.nytsolver.workers.dev/date/add/BloggingIo@7?date=2025-12-25

================================================================
AUTOMATION (CRON JOBS)
================================================================

The worker automatically fetches the latest puzzle at the following times:
- Daily: 03:01 UTC (Backup at 03:11 UTC) - Targeted at 10:01 PM EST weekday release.
- Saturdays: 23:01 UTC (Backup at 23:11 UTC) - Targeted at 6:01 PM EST Sunday release.
